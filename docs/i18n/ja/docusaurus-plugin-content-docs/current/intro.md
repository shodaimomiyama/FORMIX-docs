# FORMIX 入門

**FORMIX**（Deterministic Threshold Proxy Re-Encryption System）は、Arweave上に保存された暗号化データのための分散型アクセス制御システムです。元の暗号鍵を公開することなく、安全でパーミッションレスなデータ共有を可能にします。

## FORMIXとは？

FORMIXは**閾値プロキシ再暗号化（TPRE）**技術を実装しており、データ所有者が分散プロキシノードを通じて他のユーザーに復号権限を委任できます。これにより、以下の特性を持つトラストレスなシステムを構築します：

- **単一障害点なし** - 復号には複数のプロキシノードの協力が必要
- **所有者が制御を保持** - アクセス権はいつでも付与・取り消し可能
- **エンドツーエンド暗号化** - データはプロセス全体を通じて暗号化されたまま

## 主な特徴

### 閾値プロキシ再暗号化
暗号技術を使用して、再暗号化機能を複数の独立したノードに分割します。閾値数のノードがデータを再暗号化するために協力する必要があり、ビザンチン障害耐性を提供します。

### 分散アーキテクチャ
永続ストレージにArweave、コンピューティングにAO Networkを使用し、中央の制御点や障害点を排除します。

### マルチロールWasmアクター
3つの異なるプロセスタイプ（Owner、Holder、Requester）がAO上のWebAssemblyモジュールとして動作し、アクセス制御ワークフローにおいてそれぞれ特定の責任を担います。

## 技術スタック

| コンポーネント | 技術 | 目的 |
|-----------|------------|---------|
| ストレージ | Arweave | 永続的な分散データストレージ |
| コンピューティング | AO Network | 分散プロセス実行 |
| 暗号化 | Umbral TPRE (Rust/Wasm) | 閾値プロキシ再暗号化 |
| スマートコントラクト | AO Network (Lua) | 再暗号化鍵の管理 |

## 開発状況

FORMIXは現在活発に開発中です。コア暗号プリミティブとAOプロセスアーキテクチャは実装済みで、以下の作業が進行中です：

- 強化された鍵管理
- EVM統合のためのガス最適化
- 追加のセキュリティ監査
- 開発者ツールとSDK
